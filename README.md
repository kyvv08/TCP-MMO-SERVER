⚔️ TCP MMO Server

6400 x 6400 크기의 월드에서 평균 8,000명, 최대 10,000명의 동시 접속자를 수용할 수 있도록 설계된 TCP 기반 MMO 서버
C++로 구현되었으며, 대규모 유저 관리와 효율적인 메시지 전송을 위해 섹터 단위 캐릭터 관리와 최적화된 네트워크 구조를 적용

📌 주요 기능 (Features)

- 대규모 월드 지원

6400 x 6400 크기의 월드 맵

평균 8,000명, 최대 10,000명 동시 접속 처리

- 섹터 단위 관리

월드를 섹터로 분할하여 캐릭터 관리

섹터 단위로 메시지 전송 범위를 제한 → 네트워크 부하 감소

- 세션(Session) 관리

TCP 소켓 프로그래밍 기반

연결/해제 관리 및 안정적인 세션 유지

- 메시지 처리

직렬화/역직렬화 과정에서 직렬화 버퍼 & 링 버퍼 활용

직렬화 버퍼 메모리 풀 적용으로 메모리 할당/해제 비용 최소화


📖 학습 포인트 (What I Learned)

대규모 MMO 서버 구조 설계 경험

섹터 단위 최적화를 통한 네트워크 성능 개선 방법

메시지 직렬화/역직렬화 및 메모리 관리 기법 (메모리 풀, 링 버퍼)

TCP 기반 소켓 프로그래밍 심화
